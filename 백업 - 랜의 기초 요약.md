LAN (김태준 전임강사님의 자료를 요약한것.)

[] LAN의 정의와 특징

*. 중간노드의 교환이 필요없는 점대점(Point to Point) 공유물리적매체 사용
*. 공유매체의 접근제어를 위한 MAC계층 필요
*. 하나의 조직에 의해 동작되고 사용됨


[] 매체 접근 제어 (Medium Access Control)

*. 연결된 모든국(단말)이 동등한 제어기능을 갖는 분산제어방식
*. 돌발적인 트래픽에 효과적인 성능을 나타내는 '비동기'적 매체접근방식 이용
*. 대역폭 분배방식 종류 : Round-robin, reservation, contention
*. 대표적 MAC프로토콜로는 CSMA/CD, Token Ring/Token Bus가 있다.


() 대역폭 분배방식의 종류

1. Round-robin (분배)
*. 공유매체에 접속된 모든국이 순서대로 전송할 기회를 주는 방식
*. 전송기회를 잡은 국은 제한된 데이터량 또는 제한된 시간동안 매체 점유
*. 전송기회 분배 (중앙집중제어/폴링방식, 분산제어/토큰방식 사용)
*. IEEE 802.4(Token Bus), IEEE 802.5(Token Ring)에 적용된다.

2. Reservation (예약)
*. 전송할 데이터가 있을때 예약후 대역폭 사용
*. 타임슬롯 형태로 구성된 버스나 링 토폴로지에 주로 적용
*. 중앙집중/분산된 방식으로 대역폭을 예약
*. 등시성(계속 같은)을 요구하는 스트림 트래픽에 효과적이다.
*. DODB(IEEE 802.6)에 적용된다.

3. Contention (경젱)
*. 돌발적 트래픽이 많은곳에 효과적
*. 집중기능이 없어 분산된 통신방식에 매우 적합하다.
*. 복잡한 제어기가 필요없어서 구현이 매우 간단하다.
*. 부하 가중시 충돌에 의해 성능이 급격히 저하된다.
*. 통신망에서 데이터 전달 완료시간의 예측이 불가능하다.
*. 접속단말수가 적고, 낮은 부하와 짧은거리에 주로 사용한다.
*. Bus구조에 유리한 분배방식이다.
*. IEEE 802.3(CSMA/CD)에 적용된다
-> 반송파 감지 및 충돌검출 기반의 매체 접근방식

() LAN의 분류

* 물리계층의 전송방식에 따른 분류
Baseband - 디지털 자료, 변조하지 않으므로 빠르다, 데이터 전송시 사용
Broadband - 변복조가 필요하며, 음성/영상전송시 사용한다.

* 토폴로지에 의한 분류
버스 - 저럼하며, 고장이 적다, 단거리, 흔히 말하는 이더넷
가장많이 사용
링 - 먼거리가 가능, 노드 추가/변경/고장시 문제발생이 쉽다
토큰링,FDDI에 적용
성형 - 전용매체용, 높은 대역폭, LAN스위치 기반의 고속LAN에 적용된다.
----------------------------------------------------------------------------

Ethernet - IEEE 802.3 CSMA/CD프로토콜을 적용하는 Lan으로서 가장 널리 사용.
ATM - Asynchronous Transfer Mode / 초고속통신망에 사용된 교환 기법

* LAN의 실제 유효한 데이터속도는 하드웨어속도의 10-30%임 (?)

() LAN의 유형

* 공유형 - 공용매체기반, 한번에 한 단말만 Data전송 가능
성능이 나쁘지만 간단하고 저렴하여 보급형으로 사용

* 교환형 - 전용매체기반으로 중앙에 데이터교환기가 있어 동시에 모든 단말이
데이터를 전송할수 있어 대용량 LAN구축에 사용된다.

[] ISO 7계층의 구성

*. 물리계층 (Physical Layer) - 최 하위 계층 / 단위-비트
: 인접장치간 비트전송경로를 DataLink계층에 제공,
기계사양등의 전기회로, 비트전송의 절차적 수단 제공

*. Data Link Layer / 단위-프레임
인접장치간 데이터 전달기능 제공
HDLL, LADB, LAPD, LAN의 MAC기능 포함

*. Network Layer / 단위-패킷
데이터교환 및 중계기능
X.25 (연결형 패킷통신), IP

*. Transport Layer / 단위-세그먼트
에러복구, 흐름제어
TCP, UDP

*. Session Layer
응용들간의 대화, 세션 제어기능 담당
세션 연결설정, 관리, 해제 기능 수행

*. Presentation Layer
데이터 표현, 압축/보안 사항 관계
MPEG2

*. Application Layer
사용자가 OSI환경을 접근할 수 있도록 하며, 분산정보서비스 제공
FTP, VT, MHS(메세지 처리 시스템)

+ TCP/IP로써 지연시간에 관대(별로 신경쓰지않는다)하고,
